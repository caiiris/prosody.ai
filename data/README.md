# Data

- **PoetryFoundationData.csv** — Original Poetry Foundation poems (Title, Poem, Poet, Tags).
- **PoetryFoundationData_with_year.csv** — Same columns plus **Year** (poet birth year + 30 as proxy). All poems are kept; poems whose poet has no mapping will have blank `Year`. Generated by the flow below.
- **poet_year_mapping.csv** — One row per poet (`Poet`, `Year`). Built by `scripts/fetch_poet_years.py` from Wikipedia (birth year + 30).

## Fast: poet birth year + 30 (recommended)

Gets one year per poet from Wikipedia and assigns it to all their poems. ~1–2 hours for the full dataset (one lookup per unique poet).

```bash
pip install -r requirements.txt
python scripts/fetch_poet_years.py
python scripts/add_publication_year.py
```

1. `fetch_poet_years.py` — Fetches birth year for each unique poet from Wikipedia, adds 30, writes `poet_year_mapping.csv`. Resume-safe (uses `poet_year_cache.csv`).
2. `add_publication_year.py` — Joins poems with the mapping, writes `PoetryFoundationData_with_year.csv`; poems whose poet isn’t in the mapping get a blank `Year`.

## Slower: per-poem copyright year from Poetry Foundation

`scripts/scrape_pf_publication_year.py` uses Playwright to scrape the copyright line on each poem’s PF page. Much slower (tens of hours) but gives a year per poem. See that script’s docstring for usage.
